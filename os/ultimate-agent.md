## 1. Coding Mastery Module

**Source Wisdom**: Python Crash Course, Eloquent JavaScript, Automate the Boring Stuff, The Pragmatic Programmer, Clean Code, Refactoring, Design Patterns, Structure and Interpretation of Computer Programs

### Core Principles

#### Automation First (Automate the Boring Stuff)
- **Rule**: If you do it twice, automate it the third time
- **Application**:
  - Scrape competitor tour pricing weekly
  - Auto-generate partner outreach emails from templates
  - Calculate cohort economics with one command
  - Deploy to production with single push

#### Incremental Building (Python Crash Course)
- **Rule**: Build working software in small, testable chunks
- **Application**:
  - Prototype booking flow → test → deploy
  - Revenue calculator API → validate → integrate
  - Landing page → A/B test → optimize

#### DRY - Don't Repeat Yourself (Pragmatic Programmer)
- **Rule**: Every piece of knowledge has a single, unambiguous representation
- **Application**:
  - One source of truth: `os/products.md` for all pricing
  - Shared components across all landing pages
  - Template system for experience pages

#### Clean Code Principles (Clean Code + Refactoring)
```
Functions should do ONE thing
Names should reveal intent
Comments explain WHY, not WHAT
Refactor mercilessly—leave code cleaner than you found it
```

#### Error Handling Philosophy
- Validate at boundaries (user input, API responses)
- Fail fast, fail informatively
- Log everything for daemon monitoring
- Graceful degradation for user experience

#### Design Patterns for Scale
| Pattern | Use Case |
|---------|----------|
| Factory | Generate experience page variants |
| Observer | Real-time cohort availability updates |
| Strategy | Multiple pricing algorithms |
| Facade | Simple API for complex booking logic |

### Coding Decision Matrix

| Scenario | Action |
|----------|--------|
| New feature request | Write test first → implement → refactor |
| Bug discovered | Reproduce → isolate → fix → add regression test |
| Performance issue | Profile first → identify bottleneck → optimize surgically |
| Technical debt | Schedule refactoring sprints → never let it compound |

## 2. Website Architecture Engine

**Source Wisdom**: HTML/CSS Design and Build Websites, Learning Web Design, CSS Secrets, Don't Make Me Think, Refactoring UI, Ruby on Rails Tutorial, JavaScript: The Good Parts

### Core Principles

#### Mobile-First Responsive (Learning Web Design)
- **Rule**: Design for smallest screen first, enhance upward
- **Application**:
  - Touch targets 44px minimum
  - Cohort cards stack on mobile, grid on desktop
  - Hero images optimize per breakpoint

#### Semantic Structure (HTML/CSS)
```html
<!-- Structure mirrors meaning -->
<article class="experience">
  <header>
    <h1>Sushi Mastery</h1>
    <p class="meta">Tokyo • 30 Days</p>
  </header>
  <section class="journey">...</section>
  <section class="instructor">...</section>
  <footer class="cta">...</footer>
</article>
```

#### CSS Architecture (CSS Secrets + Refactoring UI)
- **Custom Properties**: Define once, use everywhere
```css
:root {
  --color-primary: #06b6d4;
  --glass-bg: rgba(255, 255, 255, 0.03);
  --spacing-unit: 8px;
}
```
- **Utility-First**: Tailwind approach for rapid iteration
- **Component Isolation**: Each experience modal self-contained

#### Performance Non-Negotiables
| Metric | Target | Method |
|--------|--------|--------|
| First Contentful Paint | <1.5s | Inline critical CSS |
| Largest Contentful Paint | <2.5s | Lazy-load below-fold images |
| Cumulative Layout Shift | <0.1 | Reserve space for images |
| Time to Interactive | <3s | Defer non-critical JS |

#### UX Laws (Don't Make Me Think)
1. **Clarity > Cleverness**: Labels say exactly what they do
2. **Scan, Don't Read**: Users scan—make CTAs obvious
3. **3-Click Rule**: Any destination reachable in 3 clicks
4. **Consistency**: Same patterns across all pages

#### Visual Hierarchy (Refactoring UI)
```
HERO: Big, bold, one message
↓
VALUE PROPS: Scannable cards
↓
SOCIAL PROOF: Testimonials, logos
↓
CTA: Impossible to miss
↓
DETAILS: For those who scroll
```

### Website Decision Matrix

| Element | Principle |
|---------|-----------|
| Navigation | Max 5 items, most important first |
| Forms | Progressive disclosure, inline validation |
| Images | WebP format, srcset for responsive |
| Typography | 2 fonts max, clear hierarchy |
| Color | 60-30-10 rule (primary-secondary-accent) |

## 5. Instructional Design Accelerator

**Source Wisdom**: Design for How People Learn, e-Learning and the Science of Instruction, Map It, The Accidental Instructional Designer, Make It Stick, Peak

### Core Principles

#### Action Mapping (Map It)
```
BUSINESS GOAL: 50 certified graduates per quarter
↓
WHAT MUST THEY DO: Pass certification, apply skills
↓
WHY AREN'T THEY: Lack practice, theory overload
↓
MINIMUM INFORMATION: Only what they need for action
↓
ACTIVITIES: Practice-based, realistic scenarios
```

#### Multimedia Learning Principles (e-Learning Science)
| Principle | Application |
|-----------|-------------|
| Coherence | Remove decorative content—only essential |
| Signaling | Highlight key points visually |
| Redundancy | Don't duplicate audio and text |
| Spatial contiguity | Words near related graphics |
| Temporal contiguity | Narration with animation |
| Segmenting | Break into digestible chunks |
| Pre-training | Teach concepts before processes |
| Modality | Graphics + narration > graphics + text |
| Personalization | Conversational tone |

#### Memory Science (Make It Stick)
- **Spaced repetition**: Review at increasing intervals
- **Interleaving**: Mix different skills in practice
- **Retrieval practice**: Test yourself, don't re-read
- **Elaboration**: Connect to existing knowledge
- **Generation**: Attempt before being told

#### Deliberate Practice (Peak)
```
1. WELL-DEFINED GOALS: Specific, measurable
2. FULL ATTENTION: No distractions
3. IMMEDIATE FEEDBACK: Know if you're right
4. STRETCH: Just beyond current ability
5. MENTAL REPRESENTATIONS: Expert thinking patterns
```

#### Experience Design Framework
```
PRE-ARRIVAL (2 weeks before)
├── Welcome video from instructor
├── Pre-reading materials
├── Cohort introductions
└── Mindset preparation

IMMERSION (During)
├── Morning: Skill practice
├── Afternoon: Theory/context
├── Evening: Reflection/integration
└── Weekly: Progress assessment

POST-COMPLETION (Ongoing)
├── Certification support
├── Alumni community
├── Career resources
└── Reunion events
```

#### Curriculum Architecture
| Phase | Duration | Focus | Method |
|-------|----------|-------|--------|
| Foundation | Days 1-7 | Basics + culture | Demo + guided practice |
| Development | Days 8-21 | Core skills | Deliberate practice |
| Integration | Days 22-28 | Synthesis | Real-world application |
| Mastery | Days 29-30 | Certification | Assessment + celebration |

### Instructional Design Decision Matrix

| Challenge | Solution |
|-----------|----------|
| Students overwhelmed | Segment content, remove fluff |
| Low engagement | Add interactivity, stories |
| Poor retention | Spaced practice, retrieval tests |
| Transfer failure | Realistic scenarios, application |
| Varied skill levels | Pre-assessment, adaptive paths |

## 6. Self-Mastery Core

**Source Wisdom**: Atomic Habits, Mindset, The Mountain Is You, Deep Work, Essentialism, The 4-Hour Work Week, Thinking Fast and Slow, Man's Search for Meaning

### Core Principles

#### Habit Architecture (Atomic Habits)
```
CUE → CRAVING → RESPONSE → REWARD

Make it OBVIOUS: Design environment for action
Make it ATTRACTIVE: Pair with enjoyment
Make it EASY: Reduce friction to 2 minutes
Make it SATISFYING: Immediate reward
```

#### Founder Daily Habits
| Time | Habit | Stack |
|------|-------|-------|
| Morning | Code for 1 hour | After coffee, before email |
| Midday | Content creation | After lunch, before calls |
| Evening | Review metrics | Before shutdown ritual |
| Weekly | Strategic thinking | Sunday morning deep work |

#### Growth Mindset (Dweck)
| Fixed Mindset | Growth Mindset |
|---------------|----------------|
| "I'm not a coder" | "I'm learning to code" |
| "This failed" | "This taught me X" |
| "They're lucky" | "They worked differently" |
| "I can't do sales" | "I'm developing sales skills" |

#### Shadow Work (The Mountain Is You)
- **Identify self-sabotage**: Scope creep, perfectionism, avoidance
- **Understand the protection**: What fear is this serving?
- **Reframe the belief**: Failure = data, not identity
- **Build antifragile**: Gain from disorder

#### Deep Work Protocol (Newport)
```
RITUAL:
- Same time, same place
- Phone off, internet blocked
- Clear objective for session
- Hard stop time

MODES:
- Monastic: Disappear for days (ideal for coding sprints)
- Bimodal: Alternate deep/shallow days
- Rhythmic: Same hours daily (most sustainable)
- Journalistic: Grab deep work when possible
```

#### Essentialism Framework
```
EXPLORE: Discern the vital few from trivial many
ELIMINATE: Cut everything that doesn't serve core mission
EXECUTE: Make execution effortless

QUESTION: "Is this the most important thing I could be doing?"
```

#### Energy Management
| Energy Type | Replenish |
|-------------|-----------|
| Physical | Sleep, exercise, nutrition |
| Emotional | Connection, gratitude, boundaries |
| Mental | Focus blocks, breaks, learning |
| Spiritual | Purpose, meaning, values alignment |

#### Decision-Making (Kahneman)
- **System 1**: Fast, intuitive, often wrong
- **System 2**: Slow, deliberate, effortful
- **Pre-mortem**: Imagine failure, work backward
- **Inversion**: What would guarantee failure? Avoid that.

### Self-Mastery Decision Matrix

| State | Action |
|-------|--------|
| Overwhelmed | Essentialism audit—cut 80% |
| Procrastinating | 2-minute rule—just start |
| Burned out | Energy audit—which type depleted? |
| Stuck | Mindset check—fixed belief? |
| Scattered | Deep work ritual—protect focus |

STRATEGIC

#### Crisis Response Protocol
```
IMMEDIATE (0-1 hour):
├── Assess severity (1-5 scale)
├── Notify key stakeholders
├── Secure safety of people
└── Preserve evidence/information

SHORT-TERM (1-24 hours):
├── Communicate with affected parties
├── Activate backup plans
├── Document everything
└── Single spokesperson

MEDIUM-TERM (24-72 hours):
├── Public statement if needed
├── Support affected parties
├── Investigate root cause
└── Legal consultation if required

LONG-TERM (1 week+):
├── Full post-mortem
├── System improvements
├── Relationship repair
├── Policy updates
```
